## 1. BPE란 무엇인가

### 한 줄 요약

> ShotGrid + Nuke 사이의 단순 반복 작업을 없애는 Beluca 전용 파이프라인 도구

### 프로그램 소개

BPE(Beluca Pipeline Engine)는 Beluca 스튜디오의 VFX 제작 환경에 최적화된 사내 전용 파이프라인 데스크톱 애플리케이션입니다. ShotGrid(프로젝트 관리 플랫폼)와 Nuke(합성 소프트웨어) 사이에서 발생하는 반복적인 수작업을 자동화하고, 스튜디오 전체가 동일한 파이프라인 규칙을 따를 수 있도록 설계되었습니다.

현재 버전은 **베타(Beta) 단계**로, Comp(합성) 파트 아티스트를 주 대상으로 운영됩니다. 기능은 지속적으로 추가·개선되고 있으며, 앞으로 FX, 3D, Matte 파트로 확장될 예정입니다. 사용 중 불편한 점이나 개선 요청은 파이프라인 담당자에게 전달해 주세요.

### 왜 만들었나

VFX 현장에서 아티스트는 매일 같은 일을 반복합니다. ShotGrid에 로그인해서 내 태스크를 확인하고, 서버 탐색기에서 최신 .nk 파일을 찾아 열고, 새 샷이 들어오면 폴더를 만들고 v001.nk를 세팅하고, 작업이 끝나면 MOV를 골라 ShotGrid에 버전을 올립니다. 이 과정을 수동으로 하면 시간이 낭비되고, 사람마다 방식이 달라 파일 구조와 네이밍이 제각각이 되어 관리가 어렵습니다. BPE는 이 모든 반복 작업을 자동화해 아티스트가 실제 작업에만 집중할 수 있는 환경을 만들기 위해 개발되었습니다.

### 어떤 도움을 주는가

| 기존 방식 | BPE 사용 후 |
|-----------|-------------|
| ShotGrid 브라우저 로그인 후 내 태스크 확인 | BPE 실행하면 내 태스크가 바로 표시됨 |
| 서버 탐색기에서 최신 .nk 파일 직접 찾기 | NukeX 버튼 한 번으로 최신 파일 자동 오픈 |
| 폴더 구조 외워서 직접 생성 | Shot Builder가 표준 폴더 + v001.nk 자동 생성 |
| ShotGrid 접속 후 수동으로 버전 업로드 | 퍼블리쉬 버튼으로 MOV 업로드 원클릭 완료 |
| 렌더 전 QC 수동 확인 | QC Checker가 렌더 전 자동으로 문제 감지 |
| 렌더 후 직접 뷰어 설정 | Post-Render Viewer가 자동으로 미리보기 생성 |

### 현재 버전 안내 및 권장 사용 방법

BPE v0.2는 베타 버전입니다. 현재 안정적으로 검증된 기능은 **My Tasks(마이테스크) 탭의 편의 기능**입니다. 샷 카드 확인, NukeX 실행, 폴더 열기, 노트 확인 등 마이테스크 중심으로 활용하시길 권장합니다.

**Shot Builder와 퍼블리쉬는 현재 베타 테스트 단계입니다.** 두 기능 모두 향후 업데이트를 통해 더욱 안정적이고 편리한 형태로 개선될 예정이므로, 정식 업데이트 이후부터 실무에 적용하시기를 권장합니다. 프리셋 매니저는 관리자 전용으로, 일반 아티스트는 읽기 전용으로 참고하시면 됩니다.

## 2. 마이테스크 (My Tasks)

### 화면 개요

프로그램을 실행하면 가장 먼저 보이는 화면입니다. ShotGrid에 등록된 내 컴프 태스크 목록을 자동으로 불러와 카드 형태로 보여줍니다. 별도로 ShotGrid 웹사이트에 접속하지 않아도 현재 진행 중인 모든 샷의 상태, 납품일, 작업 버전을 한눈에 파악할 수 있습니다.

화면은 크게 두 영역으로 나뉩니다. 왼쪽에는 샷 카드 목록이 스크롤 형태로 표시되고, 오른쪽에는 Notes 패널 또는 Shot Builder 패널이 표시됩니다. 오른쪽 상단의 탭을 클릭해 두 패널을 전환할 수 있습니다.

### 필터 사용법

화면 상단에 세 가지 필터가 있습니다. 필터를 변경하면 목록이 즉시 갱신됩니다.

**프로젝트 선택**
드롭다운에서 현재 진행 중인 프로젝트를 선택합니다. ShotGrid에 등록된 활성 프로젝트만 표시됩니다. 프로젝트를 선택하면 해당 프로젝트에서 나에게 배정된 태스크만 나타납니다.

**담당자 선택**
기본값은 로그인한 계정("나로 설정")입니다. 이름을 직접 입력하면 자동완성이 작동해 다른 팀원의 태스크를 조회할 수도 있습니다. 슈퍼바이저가 팀원 상황을 파악할 때 유용합니다.

**상태 필터**

| 상태 | 의미 | 비고 |
|------|------|------|
| 전체 | 모든 상태 표시 | 기본값 |
| wip | 작업 진행 중 | Work In Progress |
| retake | 수정 요청 | 감독/슈퍼바이저 피드백 |
| wtg | 선행 작업 대기 중 | Waiting To Go |
| fin | 작업 완료 | |
| Delivery | 납품 예정 | 납품일 기준 긴급순 자동 정렬 |

Delivery 필터를 선택하면 납품일이 가장 임박한 샷이 맨 위로 올라옵니다. 납품 일정 관리에 특히 유용합니다.

### 샷 카드 구성

각 샷은 아래와 같은 구성의 카드로 표시됩니다.

- **썸네일**: ShotGrid에 등록된 플레이트 이미지를 자동 다운로드해 표시합니다. 썸네일을 클릭하면 RV로 플레이트 MOV를 바로 재생합니다.
- **샷명 / 태스크명**: 샷 코드와 태스크 이름이 표시됩니다. (예: E107_S022_0080 / comp)
- **상태 배지**: 현재 상태를 색상으로 구분해 보여줍니다. wip=파랑, retake=빨강, fin=초록 등
- **납품일**: ShotGrid에 등록된 납품 기한을 표시합니다. 임박한 날짜는 강조 색상으로 표시될 수 있습니다.
- **VFX Work Order**: 해당 샷의 작업 지시 번호입니다. 텍스트를 클릭하면 클립보드에 복사됩니다.
- **로컬 버전**: 서버에 존재하는 현재 comp 작업의 최신 버전 번호를 자동으로 스캔해 표시합니다. (예: v007)

### 카드 버튼 사용법

각 카드에는 네 가지 액션이 있습니다.

**폴더 열기**
서버의 해당 샷 폴더(comp 작업 디렉토리)를 Windows 탐색기로 바로 열어줍니다. 서버 경로를 직접 입력하거나 탐색할 필요 없이 즉시 접근할 수 있습니다.

**NukeX 실행**
서버에서 해당 샷의 가장 최신 버전 .nk 파일을 자동으로 탐색해 NukeX로 열어줍니다. 버전 폴더를 직접 찾아다닐 필요가 없어 작업 시작 시간이 단축됩니다. 만약 서버에 .nk 파일이 아직 없다면 Shot Builder로 먼저 생성해야 합니다.

**퍼블리쉬**
완성된 comp MOV 파일을 ShotGrid에 버전으로 등록하는 다이얼로그를 엽니다. MOV 파일을 선택하고 설명을 입력한 후 업로드 버튼을 누르면 ShotGrid에 버전이 자동으로 생성됩니다. 자세한 내용은 4장을 참고하세요.

**썸네일 클릭**
카드의 썸네일 이미지를 클릭하면 해당 샷의 플레이트 MOV를 RV로 바로 재생합니다.

### 오른쪽 패널 — Notes (강력 추천)

**Notes 패널은 마이테스크에서 가장 적극적으로 활용하길 권장하는 기능입니다.** 왼쪽 샷 목록을 유지한 채로 피드백을 확인하고, 바로 해당 샷의 폴더를 열거나 NukeX를 실행할 수 있어 작업 흐름이 끊기지 않습니다.

Notes 패널에서는 최근 2주 동안 ShotGrid에 등록된 모든 피드백 노트를 한 곳에서 확인할 수 있습니다. 노트 목록에는 작성자, 날짜, 내용 요약이 표시됩니다.

**노트 클릭 시 샷 연동**
노트를 클릭하면 왼쪽 샷 카드 목록에서 해당 샷이 자동으로 선택·강조 표시됩니다. 어떤 샷에 대한 피드백인지 즉시 파악할 수 있으며, 해당 샷 카드의 폴더 열기, NukeX 실행 버튼을 바로 사용할 수 있습니다. 노트를 보면서 NukeX를 열고 바로 수정 작업에 들어갈 수 있어 매우 편리합니다.

**피드백 이미지 확인**
감독이나 슈퍼바이저가 노트에 첨부한 피드백 이미지를 BPE 안에서 바로 확인할 수 있습니다. 이미지가 여러 장인 경우 좌우로 넘겨가며 순서대로 볼 수 있습니다. ShotGrid 웹을 따로 열지 않아도 모든 첨부 이미지를 빠르게 검토할 수 있습니다.

**피드백 버전 MOV 재생**
노트에 연결된 컴프 버전이 있는 경우, 해당 버전의 MOV를 RV로 바로 재생할 수 있습니다. 피드백이 달린 정확한 버전의 영상을 즉시 확인해 어떤 시점의 작업물에 대한 코멘트인지 명확하게 파악할 수 있습니다.

**추천 활용법**
Notes 패널을 열어 두고 왼쪽 샷 목록을 함께 보는 것을 강력히 권장합니다. 피드백을 확인하고, 해당 샷을 클릭해 바로 NukeX를 열거나 폴더를 여는 흐름이 가장 효율적입니다. ShotGrid 웹 브라우저와 Nuke를 번갈아 오가는 기존 방식보다 훨씬 빠르게 작업에 복귀할 수 있습니다.

## 3. Shot Builder

### 기능 개요

Shot Builder는 새 샷 작업이 배정되었을 때 서버에 표준 폴더 구조를 생성하고, 프리셋 설정이 적용된 v001.nk 파일을 자동으로 만들어주는 기능입니다. 마이테스크 화면 오른쪽 패널에서 "Shot Builder" 탭을 클릭하면 접근할 수 있습니다.

폴더 구조와 파일 네이밍이 표준화되어 있어 누가 만들어도 동일한 구조를 유지할 수 있습니다. Nuke 파일에는 선택한 프리셋의 FPS, 해상도, OCIO, Read/Write 컬러스페이스 설정이 자동으로 적용되어 있어 세팅 없이 바로 작업을 시작할 수 있습니다.

### 언제 사용하나

새 샷 작업 지시(Work Order)가 내려왔을 때, 서버에 해당 샷 폴더가 아직 없는 경우에 사용합니다. 이미 폴더와 v001.nk가 존재하는 샷이라면 Shot Builder를 실행할 필요가 없습니다. 실수로 재생성을 시도하더라도, v001 폴더에 .nk 파일이 이미 있으면 Shot Builder가 자동으로 생성을 거부하여 기존 작업물을 보호합니다.

### 사용 순서

**1단계: 서버 루트 설정**
서버의 최상위 경로(프로젝트 폴더들이 모여 있는 곳)를 입력합니다. 예를 들어 `\\server\projects` 와 같은 형태입니다. 한 번 설정하면 자동으로 저장되어 다음에 열 때 그대로 유지됩니다.

**2단계: 프리셋 선택**
현재 작업하는 프로젝트에 맞는 프리셋을 선택합니다. 프리셋에는 해당 프로젝트의 FPS, 플레이트 해상도, OCIO 경로, 컬러스페이스 설정 등이 저장되어 있습니다. 프리셋 목록에 없다면 파이프라인 담당자에게 문의하세요.

**3단계: 샷 이름 입력**
Work Order에 기재된 샷 코드를 입력합니다. 예: `E107_S022_0080`. 샷 이름은 폴더 경로와 파일 이름에 그대로 사용되므로 정확하게 입력해야 합니다.

**4단계: 생성 클릭**
"생성" 버튼을 누르면 자동으로 다음 작업이 수행됩니다.
- 서버에 표준 폴더 구조 생성 (plate, edit, comp/devl/nuke/v001, renders, element 등)
- 해당 위치에 `E107_S022_0080_comp_v001.nk` 파일 생성
- Nuke 파일 내부에 프리셋 설정(Root, Read, Write 노드) 자동 적용

### 생성되는 폴더 구조

Shot Builder가 생성하는 표준 폴더 구조는 아래와 같습니다.

```
{서버루트}/{프리셋명}/04_sq/{EP}/{SHOT}/
    plate/         플레이트 소스 파일
    edit/          편집 레퍼런스
    comp/
        devl/
            nuke/
                v001/   {SHOT}_comp_v001.nk  ← 자동 생성
    renders/       렌더 출력 폴더
    element/       VFX 소스 엘리먼트
```

### 주의사항

- v001 폴더에 .nk 파일이 이미 있으면 생성하지 않습니다. 기존 파일을 보호하기 위한 의도된 동작입니다.
- v001 폴더 자체가 없는 경우, 폴더를 먼저 생성할지 물어봅니다.
- 샷 이름을 잘못 입력하면 폴더 이름도 잘못 생성됩니다. 반드시 Work Order를 확인하고 입력하세요.

### 앞으로의 개발 방향 — 원클릭 NK 생성

> **향후 주요 업데이트 예정:** 현재는 Shot Builder 패널에서 샷 이름을 직접 입력해야 합니다. 앞으로는 마이테스크 샷 목록의 카드에서 버튼 한 번으로 NK 생성이 완료되는 방식으로 개선할 예정입니다. 샷 카드에 이미 프로젝트·샷 정보가 모두 연결되어 있기 때문에, 별도로 정보를 입력할 필요 없이 클릭 한 번이면 표준 폴더와 v001.nk가 즉시 생성됩니다. 이 업데이트가 완료되면 Shot Builder는 훨씬 더 빠르고 직관적인 기능이 될 것입니다.

## 4. 퍼블리쉬

### 기능 개요

퍼블리쉬는 완성된 comp 결과물(MOV 파일)을 ShotGrid에 버전으로 등록하는 기능입니다. 마이테스크 샷 카드의 "퍼블리쉬" 버튼을 클릭하면 해당 샷에 연결된 업로드 다이얼로그가 열립니다. ShotGrid 웹사이트에 접속하거나 별도의 업로드 도구 없이, BPE 안에서 바로 버전 등록을 완료할 수 있습니다.

업로드가 완료되면 ShotGrid에 새 버전이 생성되고, 슈퍼바이저나 클라이언트가 ShotGrid에서 바로 확인할 수 있습니다. 선택적으로 업로드와 동시에 태스크 상태를 자동으로 변경할 수도 있습니다.

### 사용 순서

**1단계: 퍼블리쉬 버튼 클릭**
마이테스크의 해당 샷 카드에서 "퍼블리쉬" 버튼을 클릭합니다. 버튼을 누르면 해당 샷 정보(샷명, 태스크)가 자동으로 채워진 다이얼로그 창이 열립니다.

**2단계: MOV 파일 선택**
업로드할 MOV 파일을 다이얼로그 창에 드래그&드롭하거나, 파일 경로를 직접 입력합니다. 일반적으로 renders 폴더의 최신 버전 MOV를 선택합니다.

**3단계: 버전 정보 확인 및 입력**
- **버전명**: 파일명 기준으로 자동 입력됩니다. 필요시 수동으로 수정 가능합니다.
- **아티스트**: 현재 로그인한 계정으로 자동 설정됩니다. 자동완성을 통해 변경할 수 있습니다.
- **태스크**: 버튼을 클릭한 샷의 태스크가 자동 연결됩니다.
- **설명**: 이번 버전에서 작업한 내용이나 특이사항을 간단히 적습니다. 슈퍼바이저가 검토할 때 참고합니다. 선택사항이지만 작성을 권장합니다.

**4단계: 업로드 후 상태 변경 옵션**
업로드가 완료된 후 태스크 상태를 자동으로 변경할 수 있습니다. 예를 들어 wip 상태에서 퍼블리쉬를 완료하면 review 상태로 자동 전환되도록 설정할 수 있습니다. 필요에 따라 활성화하세요.

**5단계: 업로드 실행**
"업로드" 버튼을 클릭하면 ShotGrid에 버전이 등록됩니다. 파일 크기에 따라 수십 초에서 수 분이 소요될 수 있습니다. 업로드 중에는 진행 상황이 표시됩니다.

### 네이밍 가이드

버전명은 팀 전체가 일관된 규칙을 따르는 것이 중요합니다. 일반적으로 `{샷명}_comp_v{버전번호}` 형식을 사용합니다. 예: `E107_S022_0080_comp_v007`. 파일명이 이 규칙을 따르고 있다면 자동 입력된 버전명을 그대로 사용하면 됩니다.

### 앞으로의 개발 방향 — 원클릭 업로드 및 타임코드 입력

> **향후 주요 업데이트 예정:** 현재 퍼블리쉬는 다이얼로그에서 MOV 파일을 선택하고 정보를 확인하는 단계가 필요합니다. 앞으로는 샷 카드에서 퍼블리쉬 버튼을 클릭하는 순간, ShotGrid에서 해당 태스크의 모든 정보(샷명, 태스크, 아티스트, 프로젝트 정보 등)를 자동으로 불러와 별도 입력 없이 즉시 업로드가 완료되는 방식으로 개선할 예정입니다. 아울러 **타임코드 입력 기능**도 추가되어, 퍼블리쉬 시 해당 버전의 시작/끝 타임코드를 함께 등록할 수 있게 됩니다. 이를 통해 ShotGrid에서 버전 검색 및 리뷰 시 더욱 정확한 편집 정보를 함께 관리할 수 있습니다.

## 5. 프리셋 매니저

### 현재 기능 개요

프리셋 매니저는 프로젝트별 Nuke 설정을 저장하고 관리하는 기능입니다. 프리셋 하나에는 해당 프로젝트에서 사용하는 FPS, 플레이트 해상도, OCIO 경로, Read 노드 입력 컬러스페이스, Write 노드 출력 포맷과 컬러스페이스 등이 담겨 있습니다. 사이드바의 "Preset Manager"를 클릭하면 접근할 수 있으며, 실수로 설정을 변경하는 것을 방지하기 위해 비밀번호 잠금이 적용되어 있습니다.

프리셋은 Shot Builder에서 v001.nk를 생성할 때 자동으로 적용됩니다. 신규 프로젝트가 시작될 때 파이프라인 담당자가 프리셋을 등록해 두면, 아티스트는 Shot Builder에서 프리셋을 선택하는 것만으로 올바른 프로젝트 설정이 적용된 Nuke 파일을 받을 수 있습니다.

### 프리셋에 저장되는 정보

| 항목 | 설명 |
|------|------|
| 프로젝트 타입 | 에피소드, 피처, CF 등 |
| 프로젝트 코드 | 서버 폴더명과 동일한 코드 |
| FPS | 프레임 레이트 |
| 플레이트 해상도 | Root 노드 포맷 |
| OCIO 경로 | 컬러 매니지먼트 설정 파일 위치 |
| Read 입력 컬러스페이스 | 플레이트 Read 노드 기본값 |
| Write 출력 포맷 | EXR, MOV 등 |
| Write 컬러스페이스 | 출력 컬러스페이스 |
| Write 채널 | RGBA, RGB 등 |
| Write 데이터 타입 | 16bit, 32bit 등 |

### 프리셋 만들기 — 방법 A: 직접 입력

왼쪽 폼에 각 항목을 직접 입력하고 프리셋 이름을 지정한 뒤 "저장" 버튼을 클릭합니다. 신규 프로젝트가 시작될 때 파이프라인 담당자가 주로 사용하는 방법입니다.

### 프리셋 만들기 — 방법 B: NK 파일에서 가져오기

이미 올바르게 세팅된 기존 .nk 파일이 있다면, 해당 파일을 분석해 자동으로 프리셋을 생성할 수 있습니다. "NK 파일에서 가져오기" 버튼을 클릭한 후, .nk 파일을 선택하고 "NK 분석하기"를 누르면 파일 내의 Root, Read, Write 노드 설정을 자동으로 추출합니다. 추출된 값을 확인한 뒤 프리셋 이름을 입력하고 "프리셋 생성"을 클릭합니다.

### 앞으로의 개발 방향 — 자동화 세팅 및 통합 매니저로 확장

현재 프리셋 매니저는 기능적으로 잘 동작하지만, 앞으로 두 가지 큰 방향으로 발전시킬 계획입니다.

**1. 프리셋 등록 자동화**
현재는 프리셋을 만들 때 각 항목을 하나하나 수동으로 입력하거나 NK 파일에서 추출해야 합니다. 앞으로는 NK 파일을 지정하면 모든 설정이 자동으로 분석·적용되는 완전 자동화 세팅 방식으로 전환할 예정입니다. 설정 항목을 직접 확인하거나 수정할 필요 없이 NK 파일 하나만 있으면 프리셋 등록이 완료됩니다.

**2. 통합 매니저(Manager)로 확장**
프리셋 관리 기능에서 나아가, 리더십·매니지먼트 역할을 지원하는 통합 매니저 기능으로 발전시킬 계획입니다. 구체적으로는 아래와 같은 기능을 목표로 합니다.
- **플레이리스트 리뷰**: TM(팀 매니저)이나 SV(슈퍼바이저) 레벨에서 올라온 버전들을 ShotGrid 플레이리스트처럼 한 화면에서 모아보고, 피드백을 바로 남길 수 있는 인앱 리뷰 환경 구축
- **샷 진행 상황 대시보드**: 아티스트별·샷별 작업 진행 상황을 한눈에 파악할 수 있는 대시보드. ShotGrid에 접속하지 않고도 팀 전체의 진행 현황을 모니터링 가능
- **리더십 매니징 도구**: 태스크 배정, 상태 일괄 변경, 납품일 기준 우선순위 정렬 등 리드 아티스트와 프로덕션 매니저가 팀을 관리하는 데 필요한 기능 추가

이 통합 매니저는 단순한 프리셋 관리를 넘어, BPE를 스튜디오 전체의 제작 관리 허브로 성장시키기 위한 핵심 방향입니다.

## 6. 툴스 (Tools)

### 기능 개요

사이드바의 "Tools"를 클릭하면 Nuke 작업 자동화 도구를 켜고 끌 수 있는 화면이 표시됩니다. 현재 두 가지 도구를 제공합니다. QC Checker와 Post-Render Viewer입니다. 각 도구는 토글 스위치로 활성화·비활성화하며, 설정을 변경한 후에는 반드시 Nuke에서 `BPE Tools → Reload Tool Hooks`를 실행해야 적용됩니다.

### QC Checker

렌더를 시작하기 전에 자동으로 품질 항목을 점검합니다. 문제가 감지되면 렌더를 중단하고 경고 메시지를 표시해 잘못된 설정으로 긴 시간을 허비하는 것을 방지합니다.

**체크 항목 예시**
- Write 노드의 출력 경로가 실제로 존재하는지 확인합니다.
- 파일 포맷과 채널 설정이 올바른지 확인합니다.
- Root 노드의 FPS와 포맷 설정을 점검합니다.
- Write 노드가 활성화 상태인지 확인합니다.

QC Checker를 토글로 켜고, Nuke에서 `BPE Tools → Reload Tool Hooks`를 실행한 뒤 렌더를 시작하면 렌더 전 자동으로 체크가 실행됩니다. 문제가 없으면 렌더가 정상적으로 진행됩니다.

### Post-Render Viewer

렌더가 완료된 직후 자동으로 결과물을 확인할 수 있는 Read 노드를 Nuke 내에 생성합니다. 렌더 결과물을 별도로 찾아 열 필요 없이, 렌더가 끝나는 순간 바로 뷰어에서 확인할 수 있습니다. 긴 렌더 후 결과를 빠르게 검토해야 할 때 유용합니다.

**동작 순서**
- 렌더 완료 시 `bpe_render_preview` 라는 이름의 Read 노드가 자동으로 생성됩니다.
- Read 노드는 방금 렌더된 시퀀스 경로를 자동으로 읽습니다.
- 이미 같은 이름의 노드가 있으면 경로만 업데이트합니다.

### 적용 방법 순서

1. BPE 앱의 Tools 탭에서 원하는 도구를 토글로 활성화합니다.
2. Nuke를 열거나, 이미 열려 있다면 Nuke 상단 메뉴에서 `BPE Tools → Reload Tool Hooks`를 클릭합니다.
3. 이제부터 해당 도구가 렌더 전/후에 자동으로 실행됩니다.
4. 설정을 변경할 때마다 Step 2를 반복해야 합니다.

### 앞으로의 개발 방향 — 아티스트 요청 기반 툴셋 확장

> **Tools 탭은 앞으로 계속 성장할 공간입니다.** 현재는 QC Checker와 Post-Render Viewer 두 가지만 제공하지만, 아티스트들이 "이런 기능이 있으면 편하겠다"고 요청하는 편의 기능들을 하나씩 툴셋 형태로 추가해 나갈 예정입니다. 각 툴은 토글로 켜고 끌 수 있는 구조로 설계되어, 필요한 사람만 선택적으로 사용할 수 있습니다. 현장에서 반복적으로 발생하는 불편함이나 자동화하고 싶은 작업이 있다면 파이프라인 담당자에게 적극적으로 요청해 주세요. 작업자의 피드백이 곧 새로운 툴로 만들어집니다.

## 7. 앞으로의 계획 (로드맵)

현재 BPE v0.2는 Comp 아티스트 중심의 베타 버전입니다. 앞으로 아래 네 가지 방향으로 순차적으로 확장해 나갈 계획입니다. 각 기능의 개발 일정은 내부 일정에 따라 변경될 수 있으며, 작업자 피드백에 따라 우선순위가 조정될 수 있습니다.

### Nuke 내부 통합

현재는 BPE 앱을 별도 창으로 실행하고 Nuke를 따로 열어야 합니다. 앞으로는 BPE의 주요 기능(마이테스크, Shot Builder, 퍼블리쉬 등)을 Nuke 내부 패널로 도킹하여, Nuke를 벗어나지 않고도 모든 파이프라인 기능을 사용할 수 있게 됩니다. 작업 중 Nuke와 BPE 사이를 전환하는 번거로움이 없어집니다.

### FX / 3D / Matte 파트 통합

현재 BPE는 Comp 파트에 특화되어 있습니다. 태스크 타입, 폴더 구조, Shot Builder, 툴체인 모두 Comp 기준으로 설계되어 있습니다. 앞으로 FX, 3D, Matte 파트의 작업 방식과 폴더 구조를 분석해 각 파트에 맞는 기능을 추가할 예정입니다. 파트별 태스크 필터와 커스텀 폴더 템플릿 등이 포함됩니다.

### ComfyUI 연동 및 AI 도구 바로가기

ComfyUI(로컬 AI 이미지 처리 플랫폼)와 BPE를 연동하여, 별도의 설정 없이 인앱에서 바로 AI 기능을 활용할 수 있게 됩니다. 무료 모델 범위 안에서 배경 제거(Remove BG), Matte 생성, Depth 맵, Mask 생성 등을 지원할 계획입니다. 또한 스튜디오에서 구독 중인 AI 서비스 바로가기 패널을 추가해, 브라우저를 열지 않고 빠르게 접근할 수 있게 됩니다.

### 작업자 피드백 기반 지속 개선

BPE는 현장 아티스트의 의견을 반영해 지속적으로 개선됩니다. 인앱 피드백 창구를 통해 불편한 점이나 추가 요청을 바로 전달할 수 있게 됩니다. 새 버전이 나오면 프로그램 안에 업데이트 알림이 표시되며, 알림 창에서 "설치" 버튼을 누르면 자동으로 업데이트됩니다. 변경 내용도 버전 노트로 앱 안에서 확인할 수 있게 됩니다.

## 8. Q&A

**Q. 프로그램을 열었는데 샷이 하나도 안 나와요.**
A. 상단 필터에서 프로젝트를 먼저 선택해야 합니다. 프로젝트를 선택한 후에도 나오지 않으면, 담당자 필터가 "나로 설정"이 맞는지 확인하세요. ShotGrid에 해당 프로젝트의 태스크가 나에게 배정되어 있지 않으면 목록이 비어있을 수 있습니다. ShotGrid 연결에 문제가 있다면 파이프라인 담당자에게 문의하세요.

**Q. NukeX 버튼을 눌렀는데 아무 반응이 없거나 오류가 나요.**
A. 서버 드라이브가 마운트되어 있는지 먼저 확인하세요. 서버에 연결되지 않은 상태에서는 파일을 찾을 수 없습니다. 서버에 .nk 파일이 아직 없다면 Shot Builder로 먼저 v001.nk를 생성해야 합니다. NukeX 실행 경로 설정이 잘못된 경우에도 열리지 않을 수 있으니 파이프라인 담당자에게 확인 요청하세요.

**Q. Notes 패널에서 노트를 클릭했는데 왼쪽 샷 목록이 바뀌지 않아요.**
A. 마이테스크에서 해당 프로젝트와 담당자 필터가 올바르게 설정되어 있는지 확인하세요. 노트에 연결된 샷이 현재 필터 조건에 포함되어 있지 않으면 샷 목록에서 선택이 되지 않을 수 있습니다.

**Q. Shot Builder에서 "v001이 이미 존재합니다"라는 오류가 나요.**
A. 의도된 동작입니다. 이미 .nk 파일이 있는 폴더에 다시 생성하면 기존 작업물이 덮어쓰여질 수 있어 이를 방지합니다. 정말로 새로 생성이 필요하다면 기존 v001 파일을 백업 후 삭제하거나, 다른 버전 폴더(v002 등)에 수동으로 생성하세요.

**Q. Shot Builder로 생성한 Nuke 파일을 열었는데 컬러 설정이 이상해요.**
A. Shot Builder가 참조하는 프리셋이 현재 프로젝트와 맞는지 확인하세요. 프리셋 매니저에서 해당 프리셋의 OCIO 경로와 컬러스페이스 설정을 확인합니다. OCIO 파일이 서버에 없거나 경로가 바뀐 경우에도 이 문제가 발생합니다.

**Q. 프리셋 매니저에 들어가려면 비밀번호가 뭔가요?**
A. 비밀번호는 파이프라인 담당자에게 문의하세요. 프리셋 매니저는 아티스트가 실수로 프리셋을 수정하는 것을 방지하기 위해 잠금이 적용되어 있습니다. 일반 아티스트는 프리셋을 수정할 필요 없이 Shot Builder에서 선택해서 사용하면 됩니다.

**Q. 퍼블리쉬를 완료했는데 ShotGrid에서 버전이 안 보여요.**
A. ShotGrid 웹페이지를 새로고침하거나 잠시 기다려 보세요. 파일 크기에 따라 반영까지 약간의 시간이 걸릴 수 있습니다. 업로드 중 인터넷 연결이 끊어진 경우에는 다시 시도하세요. 반복적으로 실패한다면 파이프라인 담당자에게 문의하세요.

**Q. Tools 탭에서 QC Checker를 켰는데 Nuke에서 동작하지 않아요.**
A. BPE에서 설정을 변경한 후에는 반드시 Nuke 상단 메뉴에서 `BPE Tools → Reload Tool Hooks`를 실행해야 합니다. 이 단계를 빠뜨리면 설정이 반영되지 않습니다. Reload Tool Hooks를 실행한 후에도 동작하지 않으면 파이프라인 담당자에게 문의하세요.

**Q. 새 버전으로 업데이트하려면 어떻게 해야 하나요?**
A. 새 버전이 배포되면 BPE 앱 실행 시 상단에 업데이트 알림이 자동으로 표시됩니다. 알림 창에서 "설치" 버튼을 누르면 자동으로 다운로드 및 업데이트가 완료됩니다. 별도로 설치 파일을 받거나 실행할 필요가 없습니다. 업데이트 후에는 앱을 재시작하면 됩니다.

**Q. 프로그램이 갑자기 꺼지거나 오류 메시지가 나와요.**
A. 로그 파일을 파이프라인 담당자에게 전달하거나, 어떤 동작을 했을 때 발생했는지 재현 방법을 알려주세요. 재현 방법이 있으면 빠르게 수정할 수 있습니다.
